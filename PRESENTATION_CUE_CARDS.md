# Large Deal Report Automation - Presenter Cue Cards

Use this as a concise prompt sheet while presenting.

## 0:00-0:45 | Hook

- "This is not a coding lesson. It is a practical way to remove repetitive work."
- "Quick show of hands: who has done copy/paste reporting this week?"
- Pause, acknowledge responses, and connect to pain.

## 0:45-1:45 | Before vs After

- Before: manual steps, context switching, error risk.
- After: one-click flow, consistent output, less repetitive effort.
- Line to repeat: "Python acts like a digital assistant for repeatable admin work."

## 1:45-3:15 | 8-step Story

1. Find latest report template.
2. Download latest email attachment.
3. Update Worksheet 1.
4. Refresh Worksheet 2.
5. Create Worksheet 3 mobile copy.
6. Save date-stamped output.
7. Email distribution list.
8. Cleanup and log.

## 3:15-5:15 | Code Proof (Keep light)

- Show only short snippets.
- Translate each snippet into business value immediately.
- Avoid deep syntax explanations.

Useful line:

- "The goal is not code complexity. The goal is reliable process execution."

## 5:15-6:15 | Reliability Message

- Fallback logic exists when refresh fails.
- Logs capture what happened for troubleshooting.
- "Automated does not mean fragile; it means repeatable with safeguards."

## 6:15-7:45 | Teach-back Moments

### 4-question test

- Repetitive?
- Rule-based?
- Digital input/output?
- Time-consuming or error-prone?

If 3+ yes, it is a good automation candidate.

### Reusable blueprint

Input -> Process -> Output -> Notify -> Log

## 7:45-9:00 | Team Interaction

- Give 30-60 seconds of silence.
- Ask for two volunteers.
- For each example, quickly map against the 4-question test.

## 9:00-10:00 | Close

- "We do not need everyone to code."
- "We need everyone to identify repeatable work worth automating."
- "Action this week: each person submits one candidate task."
- Final line: "Let humans do judgment; let Python do repetition."

## Quick Q&A Backup

- Q: "Do we need advanced Python?"
  - A: "No. First skill is identifying the right task."
- Q: "Is this only for one report?"
  - A: "No. The pattern is reusable for many recurring workflows."
- Q: "What if it fails?"
  - A: "Logs and fallback behavior make issues visible and manageable."
